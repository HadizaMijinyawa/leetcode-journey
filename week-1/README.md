## Key Learnings So Far

- Big O describes how runtime/memory grows with input size.
- Access by index: O(1); Search/Insert/Delete (unsorted): O(n)
- Space complexity counts extra memory created, not the input itself.
- A loop does not automatically mean O(n) space – only if you allocate a data structure that grows with n.
